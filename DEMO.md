# デモンストレーション

このアプリケーションの動作を確認するためのデモ手順です。

## デモ環境の準備

### 必要なもの
- Webブラウザ（Chrome, Firefox, Safari, Edgeなど）
- インターネット接続
- Supabaseアカウント（無料プランで十分）

## デモシナリオ

### シナリオ1: 初回ログインとメモ作成

#### ステップ1: ログイン
1. アプリケーションを開く
2. パスコード `123456789012` を入力
3. 「ログイン」ボタンをクリック
4. **期待結果**: メモ管理画面に遷移

#### ステップ2: メモ作成
1. 「新規メモ作成」ボタンをクリック
2. タイトルを「デモメモ」に変更
3. 以下の内容を入力:
```markdown
# デモメモ

これはテストメモです。

## 機能一覧
- **Markdown対応**
- メモの作成・編集
- アクセスログ

## タスク
- [ ] 機能テスト
- [x] ログイン確認
```
4. 「保存」ボタンをクリック
5. **期待結果**: Markdownが正しくレンダリングされる

### シナリオ2: メモ編集とログ確認

#### ステップ3: メモ編集
1. 作成したメモを選択
2. 「編集」ボタンをクリック
3. 内容を追加または変更
4. 「保存」ボタンをクリック
5. **期待結果**: 変更が保存される

#### ステップ4: アクセスログ確認
1. サイドバー下部のアクセスログを確認
2. **期待結果**: 「閲覧」と「編集」のログが表示される

### シナリオ3: パスコード変更

#### ステップ5: パスコード変更
1. 「パスコード変更」ボタンをクリック
2. 新しいパスコード `999999999999` を入力
3. 確認のため再度同じパスコードを入力
4. 「変更」ボタンをクリック
5. **期待結果**: 成功メッセージが表示される

#### ステップ6: 再ログイン確認
1. 「ログアウト」ボタンをクリック
2. 新しいパスコード `999999999999` でログイン
3. **期待結果**: ログインできて、以前のメモが表示される

### シナリオ4: 複数端末テスト

#### ステップ7: 別端末からアクセス
1. スマホまたは別のブラウザでアプリを開く
2. 同じパスコードでログイン
3. メモを編集
4. **期待結果**: 両方の端末で同じメモが表示される

### シナリオ5: Markdown機能テスト

#### ステップ8: Markdown記法のテスト
新しいメモを作成して、以下のMarkdownをテスト:

```markdown
# 見出し1
## 見出し2
### 見出し3

**太字** *斜体* ~~取り消し線~~

- 箇条書き1
- 箇条書き2
  - サブ項目

1. 番号付きリスト1
2. 番号付きリスト2

[リンク](https://example.com)

> 引用文です
> 複数行も可能

インラインコード: `const x = 10;`

コードブロック:
```javascript
function hello() {
  console.log("Hello, World!");
}
```

| 列1 | 列2 | 列3 |
|-----|-----|-----|
| A | B | C |
| D | E | F |
```

**期待結果**: すべてのMarkdown記法が正しく表示される

## パフォーマンステスト

### 読み込み速度
- ページの初期読み込み: 3秒以内
- メモの切り替え: 1秒以内
- ログの更新: 即座

### 大量データテスト
1. 50個のメモを作成
2. メモ一覧がスムーズにスクロールできることを確認
3. メモの切り替えが遅延なく行えることを確認

## トラブルシューティング

### ログインできない
- Supabase設定を確認
- ブラウザのコンソールでエラーを確認
- ネットワーク接続を確認

### メモが保存されない
- Supabaseデータベースの接続を確認
- ブラウザの開発者ツールでネットワークエラーを確認

### Markdownが表示されない
- marked.jsのCDNが正しく読み込まれているか確認
- インターネット接続を確認
