## はじめに

このアプリケーションは、複数人や複数端末からメモを共有できるWebアプリケーションです。

### 主な特徴
- 📝 **Markdown対応**: 見やすいメモを作成
- 🔐 **パスコード認証**: 12桁の数字で簡単ログイン
- 👥 **複数人共有**: 同じパスコードで複数端末からアクセス
- 📊 **アクセスログ**: 誰がいつ編集したか記録
- 🔄 **リアルタイム**: 編集内容がすぐに反映

### ⚠️ 重要な注意事項
**このアプリケーションには機密情報を保存しないでください。**

以下の情報は記載しないでください:
- パスワード
- クレジットカード番号
- マイナンバー
- 個人の機密データ

機密情報を保存する必要がある場合は、別途暗号化サービスを使用してください。

## ログイン

### 初回ログイン（新規ユーザー）

1. アプリケーションのURLにアクセス
2. 12桁の数字を入力（例: `123456789012`）
   - この数字が**あなたのパスコード**になります
   - 忘れないように必ずメモしてください
3. 「ログイン」ボタンをクリック
4. 自動的にメモ管理画面に移動

### 2回目以降のログイン（既存ユーザー）

1. アプリケーションのURLにアクセス
2. 作成したパスコード（12桁）を入力
3. 「ログイン」ボタンをクリック
4. メモ管理画面に移動

### 複数端末からのアクセス

同じパスコードを使えば、複数の端末（スマホ、タブレット、PCなど）から同じメモにアクセスできます。

### チームでの共有

チームでメモを共有したい場合:
1. 一人がパスコードを作成
2. そのパスコードをチームメンバーと共有（安全な方法で）
3. 全員が同じパスコードでログイン

## メモの作成

### 新規メモの作成手順

1. メモ管理画面の左サイドバーにある「+ 新規作成」ボタンをクリック
2. 自動的に新しいメモが作成され、編集モードになります
3. タイトルを変更
   - 画面上部のタイトル欄をクリック
   - 分かりやすいタイトルを入力（例: `会議メモ 2024-11-20`）
4. 内容を入力
   - テキストエリアにメモの内容を記入
   - Markdown記法が使用可能
5. 「保存」ボタンをクリック
6. メモが保存され、閲覧モードに戻ります

### タイトルの付け方のコツ

- **日付を含める**: `2024-11-20 プロジェクト会議`
- **カテゴリを明記**: `[開発] バグ修正リスト`
- **短く具体的に**: `TODO: 今週のタスク`

## メモの編集

### 既存メモの編集手順

1. 左サイドバーのメモ一覧から編集したいメモをクリック
2. メモの内容が表示されます（閲覧モード）
3. 「編集」ボタンをクリック
4. タイトルと内容を編集
5. 「保存」ボタンで保存
   - または「キャンセル」ボタンで変更を破棄

### 編集中の注意点

- **自動保存はありません**: 必ず「保存」ボタンをクリックしてください
- **キャンセル機能**: 間違えて編集した場合は「キャンセル」で元に戻せます
- **同時編集**: 複数人が同時に編集すると、後から保存した人の内容が優先されます

## メモの削除

### メモを削除する手順

1. 削除したいメモを選択（閲覧モード）
2. 「削除」ボタンをクリック
3. 確認ダイアログが表示されます
4. 「OK」をクリックすると削除されます

⚠️ **注意**: 削除したメモは復元できません。

## Markdown記法

このアプリケーションはMarkdown形式でメモを書くことができます。

### 基本的な書き方

#### 見出し
```markdown
# 大見出し
## 中見出し
### 小見出し
```

#### 強調
```markdown
**太字**
*斜体*
~~取り消し線~~
```

#### リスト
```markdown
- 箇条書き1
- 箇条書き2
  - サブ項目

1. 番号付きリスト1
2. 番号付きリスト2
```

#### リンク
```markdown
[リンクテキスト](https://example.com)
```

#### 画像
```markdown
![画像の説明](画像のURL)
```

#### コード
```markdown
インラインコード: `const x = 10;`

コードブロック:
```javascript
function hello() {
  console.log("Hello, World!");
}
```
```

#### 引用
```markdown
> これは引用文です
> 複数行も可能です
```

#### テーブル
```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| データ1 | データ2 | データ3 |
| データ4 | データ5 | データ6 |
```

### 実用例

```markdown
# プロジェクトミーティング 2024-11-20

## 参加者
- 田中（PM）
- 鈴木（開発）
- 佐藤（デザイン）

## 議題
1. **機能追加について**
   - ユーザー認証機能を追加予定
   - 期限: 2024-12-01

2. **バグ修正**
   - ~~ログイン画面のレイアウト崩れ~~ → 修正完了
   - メモ保存時のエラー → 調査中

## タスク
- [ ] 設計書の作成（田中）
- [ ] デザインモックアップ（佐藤）
- [x] バグ調査（鈴木）

## 参考リンク
- [プロジェクト仕様書](https://example.com/spec)
- [デザインガイド](https://example.com/design)
```

## アクセスログ

### ログの見方

左サイドバーの下部に「アクセスログ」セクションがあります。

#### 表示される情報
- **アクション**: 「閲覧」または「編集」
- **メモ名**: 操作されたメモのタイトル
- **日時**: 操作された日時

#### ログの用途
- チームメンバーの活動確認
- 不正アクセスの検知
- 編集履歴の追跡

### ログの更新

- ログは10秒ごとに自動更新されます
- 最新10件のログが表示されます
- 古いログは自動的に削除される場合があります（設定による）

## パスコード変更

### パスコードを変更する理由

以下の場合はパスコードを変更することを推奨します:
- 定期的なセキュリティ更新（月1回程度）
- パスコードが漏洩した可能性がある
- チームメンバーが変わった

### 変更手順

1. 画面右上の「パスコード変更」ボタンをクリック
2. モーダルウィンドウが表示されます
3. 新しい12桁のパスコードを入力（例: `999999999999`）
4. 確認のため、もう一度同じパスコードを入力
5. 「変更」ボタンをクリック
6. 成功メッセージが表示されたら完了

### 変更後の注意

- **新しいパスコードをメモ**: 忘れるとログインできません
- **チームに共有**: 共有している場合は新しいパスコードを伝えてください
- **再ログイン不要**: すぐにログアウトする必要はありません

## よくある質問

### Q1. パスコードを忘れてしまいました

**A:** 残念ながら、パスコードを忘れた場合は復旧できません。新しいパスコードで新規ユーザーとして登録する必要があります。以前のメモにはアクセスできなくなります。

### Q2. メモが消えてしまいました

**A:** 以下の原因が考えられます:
- 誰かが削除した（アクセスログを確認）
- 間違ったパスコードでログインしている
- データベースの問題（管理者に連絡）

### Q3. 複数人で同時に編集できますか？

**A:** はい、可能です。ただし、後から保存した人の内容が優先されます。同時編集する場合は、編集する箇所を分けるか、事前に相談することを推奨します。

### Q4. スマホからも使えますか？

**A:** はい、レスポンシブデザインに対応しているため、スマホやタブレットからも快適に使用できます。

### Q5. オフラインで使えますか？

**A:** いいえ、このアプリケーションはオンラインでのみ動作します。インターネット接続が必要です。

### Q6. 画像をアップロードできますか？

**A:** 現在のバージョンでは画像の直接アップロードには対応していません。画像のURLを使用してMarkdownで表示することは可能です。

### Q7. メモをエクスポートできますか？

**A:** 現在は対応していません。必要な場合は、メモの内容をコピー＆ペーストして別のファイルに保存してください。

### Q8. パスコードは何桁でもいいですか？

**A:** いいえ、必ず12桁の数字である必要があります。それ以外の長さや文字は使用できません。

### Q9. メモの数に制限はありますか？

**A:** 技術的な制限はありませんが、大量のメモを作成するとパフォーマンスに影響する可能性があります。

### Q10. 安全性は大丈夫ですか？

**A:** 基本的なセキュリティ対策は実施していますが、完全に安全とは言えません。そのため、**機密情報は保存しないでください**。

## サポート

問題が発生した場合:
1. まずこのガイドのトラブルシューティングセクションを確認
2. SETUP.mdのトラブルシューティングも参照
3. それでも解決しない場合は、GitHubリポジトリのIssuesで報告

## ベストプラクティス

より効果的にアプリを使用するために:

1. **定期的なパスコード変更**: 月1回はパスコードを変更
2. **わかりやすいタイトル**: 後から見てもわかるタイトルを付ける
3. **Markdown活用**: 見やすいメモを作成
4. **こまめな保存**: 長文を書くときは途中でも保存
5. **ログの確認**: 定期的にアクセスログをチェック
6. **バックアップ**: 重要なメモは別途バックアップを取る
